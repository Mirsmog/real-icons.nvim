INTRODUCTION                                          *real-icons-introduction*

real-icons.nvim renders file and folder icons as real terminal images. It uses
Kitty Graphics Protocol Unicode placeholders, supports Ghostty and Kitty, and
can fall back to glyph icon providers such as mini.icons or nvim-web-devicons.

The project is experimental and usable today on Ghostty and Kitty. Public APIs
may change before v1.0.

It provides real image icons in terminal UI rows, SVG icon pack rasterization,
VS Code icon theme support, custom local packs, and opt-in integrations for
popular Neovim UI plugins.

==============================================================================
REQUIREMENTS                                          *real-icons-requirements*

- Neovim 0.10+
- Ghostty or Kitty
- termguicolors
- ImageMagick for SVG icon packs and color transforms
- curl and tar for pack installation

tmux requires passthrough and RGB color support:
>
  set -g default-terminal "tmux-256color"
  set -gq terminal-overrides[1] "*:Tc"
  set -gq terminal-features[3] "xterm-ghostty:RGB"
  set -g allow-passthrough on
<

==============================================================================
SETUP                                                        *real-icons-setup*

Example lazy.nvim spec:
>
  {
    "Mirsmog/real-icons.nvim",
    build = ":RealIconsInstallPack material",
    opts = {
      pack = "material",
      integrations = {
        telescope = true,
      },
    },
  }
<

Full configuration:
>
  require("real-icons").setup({
    pack = "material",
    packs = {},
    overrides = {},
    backend = "auto",
    size = {
      cols = 2,
      rows = 1,
      pixels = 64,
      padding = 0,
      trim = false,
    },
    color = {
      tint = nil,
      saturation = 0,
      brightness = 0,
      hue = 0,
      monochrome = false,
    },
    fallback = {
      enabled = true,
      provider = "auto",
    },
    integrations = {
      bufferline = false,
      lualine = false,
      mini_files = false,
      neo_tree = false,
      nvim_tree = false,
      oil = false,
      snacks_picker = false,
      telescope = false,
      telescope_file_browser = false,
    },
  })
<

Icon sharpness:
>
  require("real-icons").setup({
    size = {
      cols = 2,
      rows = 1,
      pixels = 64,
      padding = 0,
      trim = false,
    },
  })
<

backend = "auto" detects Kitty Graphics Protocol compatible terminals.
backend = "kitty" forces the Kitty Graphics Protocol renderer, and
backend = "disabled" forces glyph fallback.

cols and rows reserve terminal cells. rows currently must be 1. pixels controls
the generated PNG size. If a pack has too much transparent padding, set
trim = true. If icons look too large after trimming, add padding.

Color transforms are applied while building the PNG cache:
>
  require("real-icons").setup({
    color = {
      saturation = -100,
    },
  })
<

Use tint to replace icon colors while preserving alpha:
>
  require("real-icons").setup({
    color = {
      tint = "#89b4fa",
    },
  })
<

brightness, saturation, and hue are percentage offsets where 0 is neutral.
Aliases are also accepted: lightness for brightness, mask and mask_color for
tint, and grayscale for monochrome. A string value such as color = "#89b4fa"
is treated as a tint.

==============================================================================
ICON PACKS                                                  *real-icons-packs*

Install the default Material Icon Theme pack:
>
  :RealIconsInstallPack material
<

The plugin stores installed packs under stdpath("data")/real-icons/packs and
generated PNG files under stdpath("cache")/real-icons.

Switch packs at runtime:
>
  :RealIconsUsePack material
  :RealIconsUsePack builtin
<

List configured packs:
>
  :RealIconsPacks
<

Lua API:
>
  require("real-icons").use_pack("material")
<

Switching packs clears uploaded terminal image ids, clears existing real-icons
extmarks, redraws statusline and tabline UI, and emits:
>
  User RealIconsPackChanged
<

Custom VS Code icon theme:
>
  require("real-icons").setup({
    pack = "flow",
    packs = {
      flow = {
        type = "vscode",
        path = vim.fn.expand("~/.vscode-oss/extensions/thang-nm.flow-icons-2.0.3"),
        theme = "flow-deep",
        license = "personal",
      },
    },
  })
<

Simple local icon folder:
>
  require("real-icons").setup({
    pack = "my-icons",
    packs = {
      ["my-icons"] = {
        type = "simple",
        path = "~/icons",
        file = "file.svg",
        folder = "folder.svg",
        extensions = {
          lua = "lua.svg",
          ts = "typescript.svg",
        },
        filenames = {
          ["package.json"] = "nodejs.svg",
        },
        folders = {
          src = "folder-src.svg",
        },
      },
    },
  })
<

Overrides:
>
  require("real-icons").setup({
    overrides = {
      extensions = {
        lua = "~/icons/custom-lua.svg",
      },
      filenames = {
        [".env"] = "~/icons/env.svg",
      },
      folders = {
        node_modules = "~/icons/node_modules.svg",
      },
    },
  })
<

INTEGRATIONS                                        *real-icons-integrations*

All integrations are opt-in.

Telescope core file pickers:
>
  require("real-icons").setup({
    integrations = {
      telescope = true,
    },
  })
<

This affects Telescope file pickers that use telescope.make_entry.gen_from_file,
including oldfiles, find_files, and git_files.

telescope-file-browser.nvim:
>
  require("telescope").setup({
    extensions = {
      file_browser = {
        disable_devicons = true,
        entry_maker = require("real-icons.integrations.telescope_file_browser").entry_maker,
      },
    },
  })
  require("telescope").load_extension("file_browser")

API                                                            *real-icons-api*

Get an icon:
>
  local icons = require("real-icons")
  local text, hl, is_default, meta = icons.get("file", "init.lua")
<

The category is explicit by design; path-only shorthand is not supported.

Supported categories:
>
  icons.get("file", "init.lua")
  icons.get("directory", "src")
  icons.get("extension", "lua")
  icons.get("filetype", "lua")
<

icons.icon() and icons.get_icon() are aliases for icons.get().

For text-based hooks, use segment() when width or metadata is needed:
>
  local icons = require("real-icons")
  local segment = icons.segment("file", "init.lua")
  return segment.text, segment.hl
<

Render an icon into a buffer-based UI:
>
  icons.render(bufnr, row, col, "file", "init.lua")
<

Advanced integrations can resolve the engine object directly:
>
  local icons = require("real-icons")
  local icon = icons.resolve("file", "init.lua")
  icons.render(bufnr, row, col, icon)
<

List configured mappings:
>
  local icons = require("real-icons")
  vim.print(icons.categories())
  vim.print(icons.list("extension"))
<

Check capabilities:
>
  local icons = require("real-icons")
  if icons.is_supported() then
    print(icons.backend())
  end
  vim.print(icons.capabilities())
<

Switch icon packs:
>
  local icons = require("real-icons")
  icons.use_pack("material")
  print(icons.pack())
  vim.print(icons.available_packs())
<

==============================================================================
COMMANDS                                                  *real-icons-commands*

:RealIconsHealth
  Run health checks.

:RealIconsDemo
  Open a demo buffer.

:RealIconsInstallPack {name}
  Install an icon pack. Currently supported: material.

:RealIconsBuildCache
  Convert icons in the active pack to cached PNG files.

:RealIconsClearCache [name]
  Clear generated PNG cache.

:RealIconsUsePack [name]
  Switch the active icon pack, or show the current pack.

:RealIconsPacks
  List configured icon packs.

:RealIconsOilEnable
  Attach the Oil integration to the current buffer.

==============================================================================
TROUBLESHOOTING                                    *real-icons-troubleshooting*

Run :RealIconsHealth first. It checks terminal support, termguicolors,
ImageMagick, tmux passthrough, and the active icon pack.

If icons do not appear:
- Confirm you are running inside Ghostty or Kitty.
- Confirm termguicolors is enabled.
- Install the default pack with :RealIconsInstallPack material.
- Run :RealIconsDemo outside tmux, then inside tmux.
- In tmux, confirm set -g allow-passthrough on is loaded.

If icons are blurry:
- Keep SVG packs as sources when possible.
- Increase size.pixels.
- Use cols = 2 for one-row icons.
- Clear the cache after changing size or color options.
